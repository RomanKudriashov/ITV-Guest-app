# E2E (Playwright)

Тесты гоняются поверх **уже поднятого** стенда — они не поднимают своё
окружение. Смысл именно в этом: проверяется та сборка, которую видит
разработчик, вместе с настоящими бэкендом, Postgres и MinIO.

```bash
docker compose up -d          # из корня репозитория
cd e2e
npm install
npm run install-browsers      # один раз: скачает chromium
npm test
```

Полезное:

```bash
npm run test:headed           # с видимым браузером
npx playwright test --debug   # пошаговая отладка
npm run report                # HTML-отчёт после запуска
E2E_BASE_URL=http://localhost:5183 npm test
```

Тесты последовательны (`workers: 1`) намеренно: они пишут в каталог общего
демо-отеля, и параллельный запуск устроил бы гонку за порядок сортировки.
Каждый тест создаёт позиции с уникальным суффиксом и убирает их за собой.

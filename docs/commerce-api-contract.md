# Контракт коммерции

Суммы считаются правильно и **фиксируются снимком в заказе**. Оплату не
подключаем — деньги не двигаются, платёжный шов остаётся заглушкой. Все
величины — в минимальных единицах (`currency_minor_units` — показатель степени).

Контракт — до кода.

## Принцип снимка

Как цены и модификаторы уже фиксируются в `OrderItem`, все **начисления**
фиксируются в `Order` на момент оформления. Отель поменял сбор с 10% на 12% —
уже оформленные заказы не меняются. Снимок хранит и итоговые суммы, и **ставки/
флаги**, по которым они получены (аудит и пересчёт не нужны — читаем готовое).

## Настройки (на существующих тенант-таблицах, новых таблиц нет)

По умолчанию всё **выключено/ноль** — поведение существующих заказов не
меняется, пока отель не включит коммерцию в CMS.

| Уровень | Поле | Смысл |
|---|---|---|
| Hotel | `service_fee_bp` | сервисный сбор, базисные пункты (1000 = 10.00%), 0 = выкл |
| Hotel | `tax_bp` | налог/НДС, б.п.; 0 = выкл |
| Hotel | `tax_inclusive` | налог уже в цене (true) или сверху (false) |
| Hotel | `tip_presets` | пресеты чаевых в процентах, напр. `[5,10,15]` |
| Hotel | `free_delivery_threshold_minor` | порог бесплатной доставки; null = нет |
| Hotel | `price_round_to_minor` | округление итога к кратному (напр. 100 = до рубля); 0/1 = нет |
| Category | `service_fee_applies` | облагается ли сбором (у еды да, у такси нет); default true |
| Category | `min_order_minor` | минимальная сумма заказа по категории; null = нет |
| ExecutionPoint | `min_order_minor` | минимальная сумма по точке; null = нет |
| Location | `delivery_fee_minor` | стоимость доставки в локацию; 0 = бесплатно |
| Item | `prep_minutes` | время приготовления/подачи, мин; null = не показывать |

## Снимок сумм в заказе (`Order`)

```
subtotal_minor        сумма позиций
service_fee_minor     сервисный сбор (только по облагаемым категориям)
tax_minor             налог (по правилу inclusive/сверху)
delivery_fee_minor    доставка (по локации, с учётом порога бесплатной)
tip_minor             чаевые (выбор гостя)
total_minor           итог к показу
charges               JSON-слепок ставок/флагов на момент оформления
```

`Order.total` (существующее поле) остаётся **итогом к показу** (`total_minor`).
Пока коммерция выключена, `total == subtotal` — как раньше.

## Расчёт (сервисный слой, чистая функция)

`compute_charges(hotel, lines, *, location, tip)` → разбивка. Порядок:

1. `subtotal` = сумма `line_total`.
2. `service_fee` = `subtotal_облагаемых_категорий * service_fee_bp / 10000`.
3. `delivery_fee` = `location.delivery_fee_minor`, но 0, если
   `subtotal ≥ free_delivery_threshold_minor`.
4. `tax`: если `tax_inclusive` — налог уже в цене, отдельной строкой 0 к итогу
   (показываем «в т.ч. НДС X»); иначе `tax = (subtotal+fee+delivery) * tax_bp/10000`.
5. `tip` = выбор гостя (пресет % от subtotal или своя сумма).
6. `total` = subtotal + service_fee + delivery + (tax если сверху) + tip,
   округлённый по `price_round_to_minor`.

Величина каждой строки и итог возвращаются вместе со ставками; заказ их
сохраняет снимком.

## Минимальная сумма

Порог = максимум из `category.min_order_minor` (по позициям заказа) и
`execution_point.min_order_minor`. Если `subtotal < порог` — оформление
блокируется: `422 order_below_minimum`, в ответе `minimum_minor` и
`shortfall_minor` (сколько не хватает), чтобы витрина показала «добавьте ещё N».

## Оформление (гость)

`POST /api/v1/guest/order` — тело получает необязательные:
```jsonc
{ "tip_minor": 0, "tip_percent": null /* одно из двух */ }
```
Ответ заказа (`serialize_order`) несёт блок `charges` (все строки снимка) и
`eta` — ожидаемое время подачи.

## Предпросчёт корзины (без создания заказа)

`POST /api/v1/guest/cart/quote` — те же входные данные, что у заказа, возвращает
разбивку сумм, порог минимума и `below_minimum`+`shortfall_minor`. Витрина
показывает строки и блокировку до оформления, ничего не создавая.

## Время подачи

`item.prep_minutes` → чип в карточке. ETA заказа = `max(prep по позициям)` +
буфер доставки; на экране статуса — «подадут к ЧЧ:ММ» (в TZ отеля). Пока у
позиций нет `prep_minutes`, чип не показываем, ETA — как раньше.

## CMS

| Метод | Путь | Назначение |
|---|---|---|
| GET/PATCH | `/api/v1/cms/commerce-settings` | сборы/налог/чаевые/доставка/округление отеля |
| PATCH | `/api/v1/cms/categories/{id}` | `service_fee_applies`, `min_order_minor` |
| PATCH | `/api/v1/cms/items/{id}` | `prep_minutes` |
| PATCH | `/api/v1/cms/locations/{id}` | `delivery_fee_minor` |

## Аналитика

Разложение выручки: агрегаты различают `revenue` (позиции), `service_fee`,
`delivery`, `tax`, `tip` — из снимка заказа. Сбор события `order.created`
несёт разбивку; редьюсер копит по строкам.

## Границы

Без реальной оплаты и фискализации. Бейджи, стартовая (активный заказ, быстрые
действия) — отдельные разделы этого контракта; промокоды/баннеры/черновики — нет.
